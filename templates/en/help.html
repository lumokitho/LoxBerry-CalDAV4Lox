Depth: Some CalDAV server are working with a calendar structur "depth 0", others with "depth 1". Select here the variant that matches your server.

URL of the calendar: Is the URL to the calendar. Most of the CalDAV servers are showing this link in the informations of the calendar.

calendar user: Is the name of the user which is necessary for CalDAV server login.

password of the calendar user: Is the password of the calendar user.

days in future: Sets how many days the appointment is passed before its activation. (Default 0).

minutes before and after: Sets how many days the appointment is passed before its activation. Example: An appointment should turn on a light in a certain period of time. After this period, this lamp is intended to be e.g. still dimmed for 10 minutes. If the date was not displayed after the end, this information could not be processed in the Miniserver. (default 60)

Google cache duration: Sets how many minutes the Google calendar query is cached on the Loxberry before the calendar is retrieved from Google again.

search terms: Sets the terms to look for in the appointment titles. Word parts are also possible. Each search term must be in a single line.

test the data: Once all necessary information has been given, the URL is displayed as it is to be entered in the virtual HTTP input in the Miniserver. On the other hand, the calendar is tested with the inputs and the output is output in the source text. Also, some command recognitions are listed, which can be used in the config.
